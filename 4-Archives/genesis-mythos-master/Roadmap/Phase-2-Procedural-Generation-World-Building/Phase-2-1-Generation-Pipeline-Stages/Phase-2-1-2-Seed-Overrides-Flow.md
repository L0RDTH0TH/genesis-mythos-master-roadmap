## Phase 2.1.2 — Seed and Override Flow

Define how DM/player seeds and overrides enter the pipeline, how they are validated, and how they are threaded through each generation stage without hardcoding narrative.

### Checklist

- [ ] Specify canonical seed schema (core fields, optional metadata, IDs, provenance)
- [ ] Define override mechanisms per stage (what can be overridden, by whom, and when)
- [ ] Describe validation rules and dry-run behavior before committing changes
- [ ] Map how accepted seeds/overrides are attached to concrete stage inputs (terrain, biome, POI, entities)
- [ ] Identify any invariants to keep seeds/overrides replayable and debuggable
