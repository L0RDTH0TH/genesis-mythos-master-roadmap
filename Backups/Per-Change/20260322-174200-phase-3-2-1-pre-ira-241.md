## Phase 3.2.1 — DM override intent envelope and regeneration gate taxonomy

**Deliverables:** First-class **`DmOverrideIntent_v0`** (name TBD) that passes the same canonicalization / schema / domain-separation discipline as player-facing intents; distinct **`RegenRequest_v0`** for structural regeneration with explicit seed, **regen_scope**, and **gate_version_id**; closed **fail-closed reason_code** rows extending the 2.2.1 pattern.

**Interfaces**

| Artifact | Role | Consumer | Ordering |
|----------|------|----------|----------|
| `DmOverrideIntent_v0` | Authoritative DM edit as replay-logged command | `IntentPlan` pipeline + tick-scoped apply | Ordered with `MutationIntent_v0` stream per [[phase-3-1-5-deterministic-agency-slice-outcomes-mutation-ledger-replay-roadmap-2026-03-22-0045]] |
| `RegenRequest_v0` | Gated subgraph replay / recompute | Stage graph + manifest boundary | Must not emit `ManifestEmit` without passing [[phase-2-2-2-intentplan-consumption-boundary-and-deterministic-verification-harness-roadmap-2026-03-20-0605]] |
| `regen_gate_version_id` | Version bump when regen outputs touch hash preimage | CI / `ReplayAndVerify` | Align with `replay_row_version` on [[phase-3-1-1-deterministic-tick-epoch-and-hash-preimage-boundaries-roadmap-2026-03-22-0015]] |

### StableMergeKey_v0 (total order for player vs DM mutation stream)

Lexicographic compare on tuple **`(tick_epoch, lane_class, mutation_sequence, canonical_intent_bytes)`**:

| Component | Definition |
|-----------|--------------|
| `tick_epoch` | Same as [[phase-3-1-1-deterministic-tick-epoch-and-hash-preimage-boundaries-roadmap-2026-03-22-0015]] — must match row under apply. |
| `lane_class` | `0` = `MutationIntent_v0` (player / agency slice), `1` = `DmOverrideIntent_v0`. **Player stream sorts before DM** so agency outcomes are applied first; DM overrides see post-player world (documented tie vs same-tick conflicts). |
| `mutation_sequence` | Monotonic per-lane counter assigned when intent enters `AgencySliceApplyLedger_v0` (aligns with [[phase-3-1-5-deterministic-agency-slice-outcomes-mutation-ledger-replay-roadmap-2026-03-22-0045]]). |
| `canonical_intent_bytes` | Canonical serialization per **2.2.1** (domain tag + sorted fields); breaks ties deterministically. |

**Determinism:** Same inputs ⇒ same sorted list; no wall-clock. If policy must change, bump `replay_row_version` / gate id and record decision id in [[decisions-log]].

### Algorithm sketch (apply order)

**Normative ordering (2026-03-22):** Gated **regen subgraph completes (or denies) before** merged player+DM ledger apply — see [[phase-3-2-2-regen-request-preconditions-and-gated-subgraph-contract-roadmap-2026-03-22-1735]] (**D-042**). DM overrides then observe **post-regen** committed buffers.

```text
function apply_tick_with_dm(world, tick_epoch, player_intents, dm_overrides, regen_requests):
  // 1) Regen first: closed subgraph success or single denial (no partial manifest)
  for r in stable_sort(regen_requests, RegenStableOrder_v0):
    outcome := run_gated_regen_subgraph(world, r)
    if outcome == FAIL:
      emit denial(REGEN_PRECONDITIONS_FAILED or REGEN_SCOPE_OVERFLOW)
      return world
  // 2) Canonicalize + validate player + DM envelopes (reject → fixed reason_code per 2.2.1)
  ordered := sort_by_StableMergeKey_v0(concat(player_intents, dm_overrides))
  // 3) Apply intents + overrides through mutation ledger → observable bundle → TickCommitRecord
  return apply_ledger_then_commit(world, tick_epoch, ordered)
```

### Fail-closed taxonomy (draft codes)

| Code | When | Replay outcome |
|------|------|----------------|
| `OVERRIDE_MANIFEST_BYPASS` | DM path skips `ManifestEmit` boundary | Fail-closed; stop consumption |
| `REGEN_HASH_CHAIN_DRIFT` | Regen output changes preimage without bump | Fail-closed |
| `REGEN_SCOPE_OVERFLOW` | regen_scope exceeds allow-list | Fail-closed denial |
| `LEDGER_ORDER_VIOLATION` | Override not comparable in policy total order | Fail-closed |

**Acceptance criteria**

- Every DM-visible world change is representable as **`DmOverrideIntent_v0`** or a documented parallel channel that still serializes into the same replay row family as tick commits.
- Regeneration never produces silent manifest drift; failures emit **exactly one** deterministic denial with stable `reason_code`.
- `handoff_readiness` **92** = normative contract sketch; **`execution_handoff_readiness` 64** until golden rows + finalized code registry.

## Research integration

### Vault-aligned synthesis (nested Research, 2026-03-22)

- **Split the channels:** DM live edits ship as **versioned override intents** ordered with tick-scoped apply (3.1.5–3.1.6); structural regeneration is a **gated replay subgraph** with explicit seed, scope, and gate version — never a silent shortcut past [[phase-2-2-2-intentplan-consumption-boundary-and-deterministic-verification-harness-roadmap-2026-03-20-0605]].
- **Extend fail-closed taxonomy like 2.2.1:** New codes (names TBD) for manifest bypass, regen precondition drift, and ledger order violations — each maps to **exactly one** deterministic denial / stop-consumption event; no free-text `reason` in replay assertions ([[phase-2-2-1-intent-canonicalization-and-denial-taxonomy-roadmap-2026-03-20-0901]]).
- **Tick commit handoff:** Only after ordered apply should **`committed_sim_observable_hash`** and **`TickCommitRecord_v0`** close; overrides that affect observables must serialize into the same **apply ledger** story as [[phase-3-1-6-tick-scoped-observable-bundle-post-apply-replay-bridge-roadmap-2026-03-22-0047]] or a documented parallel channel that still lands in the replay row ([[phase-3-1-1-deterministic-tick-epoch-and-hash-preimage-boundaries-roadmap-2026-03-22-0015]]).
- **Preimage bumps:** If an override or regen output changes manifest or tick hash preimages, bump **`replay_row_version`** / deterministic gate id per policy (Phase 3.2 stub).
