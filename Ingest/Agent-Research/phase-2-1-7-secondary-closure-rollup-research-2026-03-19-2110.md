# Phase 2.1.7 — external patterns for closure rollups

Synthesis supports the **G-P2.1-\*** rollup artifact: explicit gate outcomes, evidence links, and a single **advance** decision under `handoff_gate: true`.

## Findings

- **Stage-gate processes** separate **entrance criteria** (prerequisites to hold the review) from **exit / success criteria** (what must be true to proceed). Maps cleanly onto our **G-P2.1-\*** table + per-note evidence columns.

- **Decision outcomes** should be enumerated (**go / hold / recycle / kill** in generic PM literature). For roadmap automation we collapse to **go (advance eligible)** vs **hold (recal / deepen / wrapper)** with the decision recorded in `decisions-log` and the rollup note.

- **Evidence packages**: best practice is predetermined criteria, traceable artifacts, and owners for open issues — aligns with wiki-linked phase notes + D-013 rollup precedent for Phase 1.1.10.
