# Writing Style Guidelines

**Version: 2026-03 – post-subagent migration**

Guidelines for Second Brain subagent and pipeline documentation in this folder.

**Scope:** These guidelines apply to all documents under `Docs/`. For other Second Brain docs (e.g. in `3-Resources/Second-Brain/` such as Backbone, Queue-Sources, Parameters), follow existing conventions in README and Backbone.

---

## Purpose

Keep docs clear, professional, and scannable. Assume the reader is familiar with Obsidian and Cursor but new to this vault’s automation.

---

## Conventions

- **Structure:** Use headings (# H1 for purpose, ## H2 for sections, ### H3 for details), tables for lists and comparisons, bullet and numbered lists.
- **Mermaid:** Use flowcharts in Architecture and flow docs where they help (e.g. delegation flow).
- **Breaks:** Avoid long unbroken blocks; keep paragraphs and sections short (e.g. &lt; 12–15 lines without a subheading or list).
- **Purpose:** Start each file with a short purpose paragraph at the top.
- **Triggers and examples:** Include concrete trigger phrases and example payloads where useful.
- **Version:** Use a version note at the top: "Version: 2026-03 – post-subagent migration".
